# 医药翻译 PRD — 补充章节（结果展示 / 下载 / 历史 / 体验）

> 对应主 PRD 第 3.4、3.5、3.6 章节的落地补充。已与前后端设计对齐：单文件模式（≤100MB）、双语对照版左右并排预览、删除历史即删除存储桶文件、LLM 使用通义千问免费模型，并采用 v1 默认决策闭合未明确需求。

---

## 3.4 结果展示与下载

### 需求描述
- 支持文本翻译与文件翻译的结果展示；文件翻译需提供双语对照预览和纯译文 PDF 下载。

### 交互与验收
1) 文本翻译  
   - 展示区采用卡片或分栏，显式标注“原文”/“译文”，无对话气泡。  
   - 提供“复制译文”按钮，点击后提示“已复制”；失败提示“复制失败，请重试”。  
   - 翻译状态：加载动画/成功/失败；失败态提供“重试”按钮。

2) 文件翻译（单文件）  
   - 完成后展示文件名、完成时间、源/目标语言。  
   - 下载按钮仅保留 `纯译文 PDF`，命名规范 `<原文件名>_<目标语言>_<日期>.pdf`。  
   - 不提供原始上传文件的重新下载。  
   - 双语对照预览：左右并排（左原文、右译文），保留原页码与目录；若译文超长优先缩小字号和行距，仍放不下时再追加续页，但不修改原页页码。  
   - 文件翻译完成后默认展示双语对照预览，方便校对。  
   - 若生成失败，展示错误信息与“重试”按钮。  
   - 用户关闭页面后，若文件已进入翻译阶段，后台任务继续执行；再次进入系统时可从历史记录恢复状态和结果。  
   - 性能目标：文本翻译 `P95 ≤ 5s`；文件翻译 `10 页内 ≤ 3 分钟`。

3) 权限与安全  
   - 所有下载需鉴权接口返回流式 Blob，不暴露存储桶直链。  
   - 仅当前用户可访问其结果，直链分享无效。

## 3.5 历史记录管理

### 需求描述
- 保存文本与文件翻译记录；支持搜索、筛选、删除；删除需同步移除云端文件。

### 交互与验收
1) 列表  
   - 默认按更新时间倒序；支持按类型（文本/文件）筛选和关键词搜索（标题/文件名）。  
   - 展示摘要：标题（文本前 50 字或文件名）、更新时间、源/目标语言、状态。
   - 默认每页 20 条。

2) 查看  
   - 点击列表项右侧面板加载对应结果（文本内容或文件预览/下载按钮）。  
   - 正在进行中的文件任务：提供“取消”操作；取消后状态显示为“已取消”，但保留历史记录。  
   - 文件翻译进行中时，允许用户查看其他历史记录，也允许继续使用文本翻译。  
   - 文本翻译失败时保留失败历史记录；用户点击“重试”后更新原历史记录。  
   - 历史记录默认保留 30 天，到期后系统自动清理。

3) 删除  
   - 二次确认文案：“删除后不可恢复，将同时删除云端源文件与翻译结果。确认删除？”  
   - 确认后：调用 `DELETE /api/history/{id}`，后端需同时删除存储桶中的源文件、纯译文结果和双语预览资源；返回成功即从列表移除并清空右侧内容，上传区解锁。  
   - 若文件任务正在执行，则先取消任务再删除。

4) 分页与空态  
   - 默认 20 条/页；无数据时展示空态与“去翻译”快捷入口。  
   - 加载/错误态需明确提示并支持重试。

## 3.6 界面与体验

### 视觉与布局
- 桌面端最小分辨率 1280×720；不做移动端适配。  
- 主工作区左右两栏：左侧历史列表（可收起），右侧为翻译区域/预览。  
- 双语对照预览左右并排，标题和页码清晰对齐。

### 状态与反馈
- 统一加载/成功/失败状态样式；文件上传、翻译、下载过程均有进度或提示。  
- 上传区在有进行中文件任务时整体禁用，并提示原因；任务完成/删除后自动解锁。  
- 文件上传不支持断点续传；上传过程中若刷新页面或重新进入系统，需重新上传文件。  
- 错误边界覆盖 PDF 预览、DOCX 预览、组件级异常，提供“重试/刷新”入口。
- 若上传的 PDF 为扫描版或图片版，系统应尽早提示“暂不支持扫描版 PDF，请上传可编辑文本型 PDF 或 DOCX”。  
- 当文件页数超过 10 页时，仅提示“处理时间可能较长”，不阻止提交。

### 可访问性与可用性
- 所有操作按钮提供 hover/active 状态与可点击区域；键盘可聚焦、Enter/Space 触发。  
- 复制/下载等关键动作有即时 Toast 提示。  
- 文本输入支持 5000 字符限制的即时计数与超限提示。

## 关键对齐点汇总
- 文件上传：单文件 ≤100MB，5MB 串行分片；存在活动任务时拒绝新文件（前端禁用，后端 409）。  
- 双语对照：仅页面预览，左右并排，保留原页码/目录；纯译文 PDF 单独下载。  
- 删除历史：同步删除存储桶源文件和翻译结果；进行中任务需先取消。  
- LLM：通义千问免费模型（dashscope OpenAI 兼容），前后端需统一超时与限流策略。  

## 默认决策（v1）
- 登录：默认使用邮箱 + 密码；飞书采用 Web 应用 OAuth2 Authorization Code 模式作为可选入口，未配置时隐藏。
- 术语库：MedDRA 采用本地数据库表维护，独立版本化更新，不依赖在线公共 API。
- 数据保留：文本和文件翻译历史统一保留 30 天；系统每日自动清理过期数据。
- 任务调度：每个用户同一时间仅允许 1 个文件任务；文件翻译期间允许继续使用文本翻译和查看其他历史。
- 文件处理：仅支持 `.docx` 和 `.pdf`；扫描版 PDF 不支持；统一输出为 PDF。
- 系统范围：不做后台平台，不做管理端，不做用户自注册和密码找回功能。
